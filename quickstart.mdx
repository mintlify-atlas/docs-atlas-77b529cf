Wanman runs a supervised network of Claude Code or Codex agents on your machine. This guide walks you through cloning the repository, building the CLI, and launching your first agent matrix with `wanman takeover`.

## Prerequisites

Before you begin, make sure you have:

<Steps>
  <Step title="Install Node.js 20+ and pnpm 9+">
    Wanman requires Node.js 20 or later and pnpm 9 or later.

    ```bash
    # Enable pnpm via corepack
    corepack enable && corepack prepare pnpm@9.15.0 --activate
    ```

    Verify your versions:

    ```bash
    node --version   # must be 20.x or higher
    pnpm --version   # must be 9.x or higher
    ```
  </Step>
  <Step title="Install git">
    Git is required for worktree isolation. Verify it's installed:

    ```bash
    git --version
    ```
  </Step>
  <Step title="Authenticate Claude Code or Codex CLI">
    Wanman spawns Claude Code or Codex as agent subprocesses and uses whatever auth they already have. Install and authenticate at least one:

    <Tabs>
      <Tab title="Claude Code">
        ```bash
        npm install -g @anthropic-ai/claude-code
        claude   # follow the login prompts
        ```
      </Tab>
      <Tab title="Codex">
        ```bash
        npm install -g @openai/codex
        # Follow the Codex CLI sign-in flow
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Clone, install, and build

```bash
git clone git@github.com:chekusu/wanman.git wanman.dev
cd wanman.dev
pnpm install
pnpm build
```

`pnpm build` compiles all packages and produces the CLI at `packages/cli/dist/index.js`.

**Option A — run directly with pnpm (no PATH setup needed):**

```bash
pnpm --filter @wanman/cli exec wanman --help
```

**Option B — link globally:**

```bash
cd packages/cli
npm link
wanman --help
```

**Option C — build a standalone single-file bundle:**

```bash
pnpm --filter @wanman/cli standalone
node packages/cli/dist/wanman.mjs --help
```

## Take over a git repository

Navigate to any existing git repository and run:

```bash
cd /path/to/any/git/repo
wanman takeover .
```

What happens when you run `takeover`:

<Steps>
  <Step title="Workspace setup">
    Wanman creates `.wanman/` inside the repo and materializes a clean worktree from `HEAD`. Agents edit this worktree — your actual working tree is never touched.
  </Step>
  <Step title="Supervisor starts">
    A supervisor process starts on an auto-selected `127.0.0.1` port (default `3120`) and waits for health.
  </Step>
  <Step title="Agents spawn">
    Claude Code (or Codex) subprocesses start under an isolated `.wanman/home/` so shell profile changes stay out of your real `$HOME`.
  </Step>
  <Step title="Stream activity">
    Takeover stays in the foreground, streaming supervisor and agent activity to your terminal.
  </Step>
</Steps>

To use the Codex runtime instead of Claude:

```bash
wanman takeover . --runtime codex
```

## Talk to the matrix

Open a second terminal and interact with the running agents:

```bash
# List all registered agents and their current states
wanman agents

# Send a goal to the CEO agent
wanman send ceo "Build me a sample TODO API"

# Live-stream all supervisor and agent events
wanman watch
```

Use `--steer` to interrupt whatever the agent is currently doing:

```bash
wanman send ceo --steer "Stop — focus only on the API endpoints"
```

Read replies from an agent:

```bash
wanman recv --agent ceo
```

## Inspect outputs

Agents produce structured artifacts through `wanman artifact put`. Browse them:

```bash
wanman artifact list                # all artifacts, newest first
wanman artifact list --agent ceo    # filter by producer
wanman artifact get <id>            # full content and metadata
```

Check task progress:

```bash
wanman task list
wanman task get <task-id>
```

## Clean up

Everything wanman creates lives under `.wanman/` inside the target repo:

```
.wanman/
  worktree/   clean checkout agents edit
  home/       isolated $HOME for agent subprocesses
  agents/     per-agent workspace dirs
  wanman.db   SQLite store: messages, context, tasks, artifacts
  logs/       supervisor and agent logs
```

Stop `wanman takeover` with `Ctrl+C`, then reset all state:

```bash
rm -rf .wanman
```

Your real working tree is untouched.
